# 01 未定义行为：栈、未初始化变量与编译器警告

对应文章：《这段"简单"的C代码，竟然藏着这么多你不知道的秘密！》

## 文件
- `uninitialized_ub.c` —— 复现文章中"读取未初始化变量"的经典例子（**故意的未定义行为**，输出不可依赖）。
- `fixed_initialized.c` —— 正确写法：始终初始化局部变量，可在 `-Werror` 下零警告通过。

## 构建与运行
```bash
make run
```

## 要点
- 局部变量不初始化即读取属于**未定义行为（UB）**，"看起来对"只是巧合。
- 用 `-Wall -Wextra` 让编译器报出 `-Wuninitialized` / `-Wunused-variable`，养成零警告习惯。
